# บทนำ

## 1.1 ที่มาและความสำคัญของปัญหา

กล้องวงจรปิดเป็นอุปกรณ์ที่ถูกใช้อย่างแพร่หลายในพื้นที่สาธารณะ อาคารสำนักงาน สถานศึกษา ร้านค้า และพื้นที่ที่ต้องการเฝ้าระวังความปลอดภัย ข้อมูลจากกล้องวงจรปิดมีประโยชน์ต่อการตรวจสอบเหตุการณ์ย้อนหลัง การติดตามบุคคลที่เกี่ยวข้องกับเหตุการณ์ และการสนับสนุนการตัดสินใจของผู้ดูแลระบบ อย่างไรก็ตาม เมื่อจำนวนกล้องและระยะเวลาของวิดีโอเพิ่มขึ้น การค้นหาบุคคลจากฟุตเทจจำนวนมากด้วยแรงงานคนจะใช้เวลานาน มีโอกาสเกิดความผิดพลาด และต้องอาศัยประสบการณ์ของผู้ตรวจสอบค่อนข้างสูง

ในกระบวนการแบบเดิม ผู้ปฏิบัติงานมักต้องเปิดดูวิดีโอทีละช่วงเวลาและจดจำลักษณะของบุคคลด้วยสายตา เช่น สีเสื้อ สี/ประเภทกางเกง หรือรูปแบบการแต่งกาย วิธีดังกล่าวมีข้อจำกัดหลายด้าน เช่น ความเหนื่อยล้าของผู้ปฏิบัติงาน ความคลาดเคลื่อนจากการมองเห็น ความยากในการเปรียบเทียบภาพบุคคลจากหลายช่วงเวลา รวมถึงปัญหาคุณภาพของวิดีโอ เช่น ความละเอียดต่ำ เงา แสงสะท้อน มุมกล้องที่ไม่เหมาะสม หรือบุคคลถูกบังบางส่วน ปัญหาเหล่านี้ทำให้การค้นหาบุคคลจากกล้องวงจรปิดเป็นงานที่ใช้ทรัพยากรสูงและไม่สม่ำเสมอ

โครงงานนี้จึงมุ่งพัฒนาระบบช่วยค้นหาบุคคลจากภาพกล้องวงจรปิดโดยใช้เทคนิคคอมพิวเตอร์วิทัศน์และปัญญาประดิษฐ์ ระบบใช้แนวทางการตรวจจับแบบสองรอบ โดยรอบแรกตรวจจับบุคคลและติดตามตำแหน่งของบุคคลในวิดีโอ จากนั้นรอบที่สองตรวจจับลักษณะเสื้อผ้าและวิเคราะห์สีจากพื้นที่ของบุคคลที่ตรวจพบ ข้อมูลที่ได้ เช่น track id, bounding box, ประเภทเสื้อผ้า, สี, กลุ่มสี, เวลาในวิดีโอ และภาพครอบบุคคล จะถูกจัดเก็บเพื่อใช้ค้นหาย้อนหลังผ่านเว็บแอปพลิเคชัน

ระบบที่พัฒนาขึ้นไม่ได้มีเป้าหมายเพื่อแทนที่ผู้ปฏิบัติงานทั้งหมด แต่เป็นเครื่องมือช่วยลดภาระในการคัดกรองข้อมูลจำนวนมาก ผู้ใช้สามารถค้นหาจากเงื่อนไข เช่น ประเภทเสื้อผ้า สี ค่าความเชื่อมั่นของโมเดล หรืออัปโหลดภาพตัวอย่างเพื่อให้ระบบวิเคราะห์คุณลักษณะเบื้องต้นแล้วนำไปค้นหาในฐานข้อมูล แนวทางนี้ช่วยให้ผู้ปฏิบัติงานสามารถเริ่มค้นหาจากข้อมูลที่ถูกจัดระบบไว้แล้ว แทนการไล่ดูวิดีโอทั้งหมดด้วยตนเอง

## 1.2 ปัญหาของระบบงานเดิม

จากบริบทของการค้นหาบุคคลในวิดีโอกล้องวงจรปิด สามารถสรุปปัญหาสำคัญของระบบงานเดิมได้ดังนี้

1. การค้นหาด้วยมนุษย์ใช้เวลานานเมื่อวิดีโอมีจำนวนมากหรือมีความยาวหลายชั่วโมง
2. ผู้ตรวจสอบอาจเกิดความเหนื่อยล้า ทำให้พลาดบุคคลเป้าหมายหรือจดจำลักษณะผิดพลาดได้
3. การค้นหาจากลักษณะภายนอก เช่น สีเสื้อหรือประเภทเสื้อผ้า ยังไม่มีระบบกลางที่ช่วยจัดเก็บและกรองข้อมูลอย่างเป็นระบบ
4. วิดีโอจากกล้องวงจรปิดมีคุณภาพแตกต่างกัน ทั้งด้านแสง เงา ความละเอียด มุมกล้อง และการบดบัง ซึ่งส่งผลต่อความแม่นยำของการตรวจจับ
5. การเชื่อมโยงข้อมูลระหว่างกล้องหลายตัวเป็นงานที่ซับซ้อน และในระบบแบบเดิมมักต้องอาศัยการคาดเดาเส้นทางของบุคคลโดยผู้ปฏิบัติงาน

## 1.3 วัตถุประสงค์ของโครงงาน

วัตถุประสงค์ของโครงงานนี้มีดังนี้

1. เพื่อออกแบบและพัฒนาระบบช่วยค้นหาบุคคลจากวิดีโอกล้องวงจรปิด โดยใช้ข้อมูลลักษณะการแต่งกาย สี และตำแหน่งของบุคคลในวิดีโอ
2. เพื่อพัฒนากระบวนการตรวจจับบุคคล ตรวจจับเสื้อผ้า วิเคราะห์สี และติดตามบุคคลภายในวิดีโอ โดยใช้ YOLO11, ByteTrack และการเปรียบเทียบคุณลักษณะของภาพ
3. เพื่อพัฒนาระบบจัดเก็บผลการตรวจจับลงฐานข้อมูล PostgreSQL และจัดเก็บภาพบุคคลที่ตรวจพบผ่าน MinIO สำหรับใช้ค้นหาย้อนหลัง
4. เพื่อพัฒนาเว็บแอปพลิเคชันสำหรับค้นหาบุคคลจากเงื่อนไข เช่น ประเภทเสื้อผ้า สี ค่าความเชื่อมั่น และภาพตัวอย่าง
5. เพื่อทดสอบการทำงานของระบบในด้านความถูกต้องของกระบวนการประมวลผล ความสามารถในการจัดเก็บข้อมูล และข้อจำกัดของระบบภายใต้ทรัพยากรฮาร์ดแวร์ที่มีอยู่

## 1.4 ขอบเขตของโครงงาน

ขอบเขตของโครงงานกำหนดเพื่อให้การพัฒนาระบบอยู่ในกรอบที่สามารถดำเนินการและประเมินผลได้จริง ดังนี้

### 1.4.1 ขอบเขตด้านการประมวลผลภาพและวิดีโอ

ระบบรองรับการประมวลผลวิดีโอจากไฟล์อัปโหลดและสตรีมกล้องที่ส่งเข้าระบบผ่านช่องทางที่พัฒนาไว้ โดยกระบวนการหลักประกอบด้วยการตรวจจับบุคคล การติดตามบุคคล การตรวจจับประเภทเสื้อผ้า และการวิเคราะห์สีจากภาพครอบบุคคล ระบบใช้ bounding box จากการตรวจจับเป็นข้อมูลตั้งต้นในการประมวลผลขั้นถัดไป

### 1.4.2 ขอบเขตด้านโมเดลและคุณลักษณะที่ตรวจจับ

ระบบใช้ YOLO11 สำหรับการตรวจจับวัตถุและใช้ ByteTrack สำหรับการติดตามบุคคลภายในวิดีโอ ประเภทเสื้อผ้าที่ใช้ในระบบแบ่งออกเป็น 6 กลุ่ม ได้แก่ long_sleeve_top, short_sleeve_top, dress, skirt, shorts และ trousers การวิเคราะห์สีใช้ระบบสีที่อ้างอิงจาก HSV เพื่อช่วยลดผลกระทบจากความต่างของแสงในระดับหนึ่ง แต่ไม่สามารถแก้ปัญหาภาพที่มีแสงผิดปกติอย่างรุนแรงได้ทั้งหมด

### 1.4.3 ขอบเขตด้านการค้นหา

ผู้ใช้สามารถค้นหาบุคคลจากข้อมูลที่ระบบตรวจจับและจัดเก็บไว้ เช่น ประเภทเสื้อผ้า สี กลุ่มสี ค่าความเชื่อมั่นของโมเดล และภาพตัวอย่างที่นำเข้าสู่ระบบ ระบบจะนำภาพตัวอย่างไปวิเคราะห์คุณลักษณะเบื้องต้นเพื่อนำไปใช้เป็นเงื่อนไขในการค้นหา อย่างไรก็ตาม ระบบยังไม่ได้มุ่งเน้นการพิสูจน์ตัวตนของบุคคลแบบแน่นอน และไม่ได้ใช้การรู้จำใบหน้า

### 1.4.4 ขอบเขตด้านการติดตามบุคคล

ระบบติดตามบุคคลภายในวิดีโอหรือสตรีมเดียวเป็นหลัก และมีการใช้ feature vector ร่วมกับสีและประเภทเสื้อผ้าเพื่อช่วยลดปัญหา track id หลุดในบางกรณี การติดตามข้ามกล้องหลายตัวแบบสมบูรณ์ยังอยู่นอกขอบเขตของโครงงานนี้ แต่แนวคิดเรื่องความสัมพันธ์ของกล้องและการเตรียมข้อมูลเพื่อค้นหาข้ามกล้องถูกวางไว้เป็นแนวทางพัฒนาในอนาคต

### 1.4.5 ขอบเขตด้านระบบฐานข้อมูลและการจัดเก็บ

ระบบจัดเก็บข้อมูลการตรวจจับใน PostgreSQL เช่น รหัสวิดีโอ รหัสกล้อง track id เวลาในวิดีโอ bounding box ประเภทเสื้อผ้า สี และข้อมูลประกอบอื่น ๆ ส่วนภาพครอบบุคคลที่ตรวจพบจะจัดเก็บใน MinIO โดยไม่ได้บันทึกทุกเฟรม แต่บันทึกตามเงื่อนไขที่กำหนด เช่น บันทึกทุก ๆ ช่วงเฟรมหรือเมื่อ track id มีการเปลี่ยนแปลง เพื่อลดปริมาณข้อมูลและภาระของระบบจัดเก็บ

### 1.4.6 ขอบเขตด้านข้อจำกัดของระบบ

ระบบไม่สามารถรับประกันผลลัพธ์ที่ถูกต้องในทุกสถานการณ์ โดยเฉพาะกรณีที่ภาพมีความละเอียดต่ำมาก บุคคลอยู่ไกลจากกล้อง มุมกล้องไม่เหมาะสม มีเงาหรือแสงสะท้อนรุนแรง มีการบดบังมาก หรือเสื้อผ้ามีสีใกล้เคียงกับฉากหลัง ระบบเป็นเครื่องมือช่วยคัดกรองและสนับสนุนการค้นหา ไม่ใช่ระบบยืนยันตัวบุคคลโดยสมบูรณ์

## 1.5 ประโยชน์ที่คาดว่าจะได้รับ

1. ได้ระบบต้นแบบสำหรับช่วยค้นหาบุคคลจากวิดีโอกล้องวงจรปิดโดยใช้ข้อมูลเสื้อผ้า สี และตำแหน่งของบุคคล
2. ช่วยลดระยะเวลาในการค้นหาบุคคลจากฟุตเทจจำนวนมาก เมื่อเทียบกับการเปิดดูวิดีโอด้วยตนเองทั้งหมด
3. ช่วยจัดเก็บผลการตรวจจับให้อยู่ในรูปแบบที่ค้นหา ตรวจสอบ และนำไปวิเคราะห์ต่อได้ง่ายขึ้น
4. ช่วยให้ผู้ปฏิบัติงานสามารถใช้คอมพิวเตอร์หนึ่งเครื่องและผู้ใช้งานหนึ่งคนในการคัดกรองข้อมูลเบื้องต้นได้มีประสิทธิภาพมากขึ้น
5. เป็นพื้นฐานสำหรับการต่อยอดระบบติดตามข้ามกล้อง การวิเคราะห์เส้นทางบุคคล และการปรับปรุงโมเดลให้เหมาะสมกับสภาพแวดล้อมจริง

## 1.6 เครื่องมือและเทคโนโลยีที่ใช้

ตารางที่ 1.1 แสดงเครื่องมือและเทคโนโลยีหลักที่ใช้ในโครงงาน

| หมวดหมู่ | เครื่องมือหรือเทคโนโลยี | บทบาทในระบบ |
|---|---|---|
| Object Detection | YOLO11 | ตรวจจับบุคคลและตรวจจับประเภทเสื้อผ้า |
| Tracking | ByteTrack | ติดตามบุคคลภายในวิดีโอและกำหนด track id |
| Feature Extraction | Feature vector จากภาพบุคคล/เสื้อผ้า | ใช้ช่วยเปรียบเทียบเมื่อ track id หลุดหรือเกิดความไม่ต่อเนื่อง |
| Image Processing | OpenCV | อ่านวิดีโอ ประมวลผลเฟรม และจัดการภาพ |
| Deep Learning Framework | PyTorch | ใช้งานและทดสอบโมเดล AI |
| Backend | Python, FastAPI | ให้บริการ API และควบคุมกระบวนการประมวลผล |
| Frontend | Next.js, TypeScript | พัฒนาเว็บแอปพลิเคชันสำหรับผู้ใช้ |
| Database | PostgreSQL | จัดเก็บผลการตรวจจับและข้อมูลที่ใช้ค้นหา |
| Object Storage | MinIO | จัดเก็บภาพบุคคลที่ตรวจพบ |
| Hardware | NVIDIA GTX 1650 4GB, RAM 16GB | ใช้พัฒนาและทดสอบระบบภายใต้ทรัพยากรจำกัด |

## 1.7 ภาพรวมการทำงานของระบบ

ภาพรวมการทำงานของระบบเริ่มจากการรับวิดีโอหรือสตรีมกล้องเข้าสู่ระบบ จากนั้นระบบจะตรวจจับบุคคลในแต่ละเฟรมและสร้าง bounding box สำหรับบุคคลที่พบ เมื่อได้ตำแหน่งของบุคคลแล้ว ระบบจะครอบภาพบุคคลเพื่อนำไปตรวจจับประเภทเสื้อผ้า วิเคราะห์สี และดึงคุณลักษณะของภาพ ข้อมูลทั้งหมดจะถูกเชื่อมโยงกับ track id และเวลาของวิดีโอ ก่อนบันทึกลงฐานข้อมูลและจัดเก็บภาพที่เกี่ยวข้องใน MinIO ผู้ใช้สามารถค้นหาผลลัพธ์ผ่านหน้าเว็บโดยกำหนดเงื่อนไขหรืออัปโหลดภาพตัวอย่าง

ตารางที่ 1.2 สรุปลำดับการทำงานหลักของระบบ

| ขั้นตอน | ข้อมูลนำเข้า | ผลลัพธ์ |
|---|---|---|
| รับวิดีโอหรือสตรีม | ไฟล์วิดีโอหรือกล้อง | เฟรมภาพสำหรับประมวลผล |
| ตรวจจับบุคคล | เฟรมภาพ | bounding box ของบุคคลและค่าความเชื่อมั่น |
| ติดตามบุคคล | ผลตรวจจับบุคคล | track id และตำแหน่งต่อเนื่องของบุคคล |
| วิเคราะห์เสื้อผ้า | ภาพครอบบุคคล | ประเภทเสื้อผ้าและค่าความเชื่อมั่น |
| วิเคราะห์สี | พื้นที่เสื้อผ้า/บุคคล | สีหลัก กลุ่มสี และข้อมูลสีประกอบ |
| บันทึกข้อมูล | ผลการประมวลผลทั้งหมด | ข้อมูลใน PostgreSQL และภาพใน MinIO |
| ค้นหาย้อนหลัง | เงื่อนไขจากผู้ใช้หรือภาพตัวอย่าง | รายการบุคคล/เหตุการณ์ที่ตรงเงื่อนไข |

## 1.8 นิยามศัพท์เฉพาะ

| คำศัพท์ | ความหมายในโครงงานนี้ |
|---|---|
| Bounding Box | กรอบสี่เหลี่ยมที่โมเดลตรวจจับได้ ใช้ระบุตำแหน่งของบุคคลหรือเสื้อผ้าในภาพ |
| Track ID | หมายเลขที่ระบบติดตามกำหนดให้บุคคลในวิดีโอ เพื่อเชื่อมโยงตำแหน่งของบุคคลระหว่างเฟรม |
| Feature Vector | ค่าตัวเลขที่แทนคุณลักษณะของภาพ ใช้ช่วยเปรียบเทียบความคล้ายกันของบุคคลหรือเสื้อผ้า |
| Similarity Matching | กระบวนการเปรียบเทียบความคล้ายกันของข้อมูล เช่น feature vector สี หรือประเภทเสื้อผ้า |
| Color Group | กลุ่มสีที่ระบบจัดหมวดหมู่จากผลวิเคราะห์สี เพื่อให้ค้นหาได้ยืดหยุ่นกว่าการระบุสีเดียวแบบละเอียด |
| Confidence | ค่าความเชื่อมั่นของโมเดลต่อผลการตรวจจับหรือจำแนกประเภท |
| Re-identification | แนวคิดในการระบุตัวบุคคลเดิมจากภาพต่างช่วงเวลา โดยใช้คุณลักษณะของภาพ ไม่ใช่การยืนยันตัวตนด้วยใบหน้า |

## 1.9 โครงสร้างรายงาน

รายงานโครงงานนี้แบ่งเนื้อหาออกเป็น 5 บท ดังนี้

บทที่ 1 บทนำ กล่าวถึงที่มาและความสำคัญของปัญหา วัตถุประสงค์ ขอบเขต ประโยชน์ที่คาดว่าจะได้รับ เครื่องมือที่ใช้ และภาพรวมการทำงานของระบบ

บทที่ 2 หลักการและทฤษฎีเบื้องต้น กล่าวถึงแนวคิดที่เกี่ยวข้องกับระบบ เช่น Object Detection, Person Re-identification, Tracking, Feature Extraction, Similarity Matching, Dataset, การเลือกใช้เทคโนโลยี และประเด็นด้านกฎหมาย/จริยธรรม

บทที่ 3 วิธีดำเนินโครงงาน อธิบายการออกแบบระบบ สถาปัตยกรรม กระบวนการพัฒนาโมเดล การประมวลผลวิดีโอ การจัดเก็บข้อมูล และการออกแบบส่วนติดต่อผู้ใช้

บทที่ 4 ผลการดำเนินงานและการทดสอบระบบ นำเสนอผลการพัฒนา ผลการทดสอบระบบ ผลการประมวลผลวิดีโอ ปัญหาที่พบ และการวิเคราะห์ข้อจำกัดของระบบ

บทที่ 5 สรุปผลและข้อเสนอแนะ สรุปผลการดำเนินงานตามวัตถุประสงค์ อภิปรายข้อจำกัด และเสนอแนวทางการพัฒนาต่อในอนาคต
